**C语言重要提醒：**

**虚拟内存：**一种内存管理技术。它让程序认为自己拥有一个连续且巨大的内存空间，而实际上，这个空间是由一部分物理内存（RAM）和一部分硬盘空间共同组成的。虚拟内存 = 物理内存 + 硬盘交换空间 + 地址映射表

·进程间隔离更容易
·方便解决内存碎片问题
·方便程序员管理内存

**内存地址：**每一个存储单元的唯一编号。

**变量地址：**是一个功能说法，变量如果有多个字节，得到变量地址的时候都是首地址。

```C
char ch='1';//占1字节
char *p_ch=&ch;//占4字节（x86系统32bit）或8字节（x64系统64bit）
```

**指针数组：**
是个数组，里面装着指针。

```C
char *planets[3] = {
    //[]上来先说明planets是个数组，*再说它里面装的是指针（char*）
        "Mercury",
        "Venus",
        "Earth"
    };
int *ptr_arr[3] = {&a, &b, &c}; // 将三个独立变量的地址存入数组
```

**数组指针：**
一个指向数组的指针，括号强制 `ptr_to_arr` 先与 `*` 结合，这直接定义了 `ptr_to_arr` 是一个**指针**，跳出括号后看到 `[5]`，说明这个指针指向的是一个包含 5 个整数的**数组整体**。**结果：** 这是一个指向特定大小数组的“单体指针”。

```c
int (*ptr_to_arr)[5]=&arr;
//*首先说明它是个指针，[]再说明它是个指向数组的指针（int[5]）
```

​	*这里 `&arr` 非常关键：*

- *`arr`（数组名）：在大多数表达式中会退化为指向**第一个元素**的指针，类型是 `int *`。*
- *`&arr`（取数组地址）：获得的是指向**整个数组**的指针，类型才是 `int (*)[5]`。*

​	***如果不加括号，编译器会报错：** 因为它无法将一个 `int (*)[5]`（数组指针）	赋值给一个 `int *[5]`（数组名代表首地址，且数组本身不可被这样整体赋	值）。*

**二维数组：**
装一维数组的数组。二维数组的数组名代表一个数组指针，这个指针+1，地址会增加一个一维数组的大小。

```C
//以下两句等价
*(*(matrix+2)+3);//数组指针+2，解引用得到数组，+3后再解引用，得整型
matrix[2][3];//实际上，同上
```

看const的右边跟着什么，说明什么不能动。

```C

```

**字符串：**
在C语言中，字符串是由字符组成的序列，以空字符**'\0'终止**。
但C语言并没有专门的字符串类型，通常使用**字符数组**来存储表示字符串。

```C
//不是字符串，只是字符数组，字符'0'的ASCII码值为48
char str1[5]={'h','e','l','l','0'};
//是字符串，'\0'的ASCII码值对应为数字0
char str2[5]={'h','e','l','l','\0'};
//是字符串，数字0对应的ASCII码是'\0'
char str3[5]={'h','e','l','l',0};
//是字符串，长度为6，所以最后一个未初始化的地方会初始化为数字0
char str4[6]={'h','e','l','l','0'};
```

字符串字面值的存储位置在**只读数据段**。

**自动存储期限**（局部变量）：
存储在栈帧中，函数调用栈存在则存在，函数调用栈销毁则销毁。

**静态存储期限**（全局变量，静态全局变量，静态局部变量）：
存储在数据段，全局/静态存储区 (Global/Static Area)，生命周期过长，从程序开始到程序结束一直存在，占用内存太久

**动态存储期限**（动态变量/堆变量）：
存储在堆区，生命周期可以自己控制，想申请时申请，想释放时释放

**红黑树相比平衡二叉树的优势：**
	1.**更低的维护成本（旋转次数少）**
	这是红黑树在实际应用中最核心的优势。
	AVL 树：要求任何节点的左右子树高度差（平衡因子）不超过 1。这种严苛的限制导致每次插入或删除节点时，极易	破坏平衡，从而引发频繁的旋转操作。
	红黑树：通过五大性质确保从根到叶子的最长路径不超过最短路径的两倍。这种“弱平衡”意味着它对插入和删除操作	的包容度更高。插入操作：两者最多都只需要 2 次旋转。删除操作：AVL 树可能需要 $O(\log n)$ 次旋转，而红黑树	最多只需要 3 次旋转即可恢复平衡。
	2.**更高的插入与删除效率**
	在频繁进行增删操作的场景下，红黑树的性能表现更优。由于红黑树减少了调整结构的次数，其整体执行效率通常高	于 AVL 树。虽然红黑树的搜索深度可能略深于 AVL 树（因为平衡性稍差），但在现代计算机的处理速度下，这种微	小的搜索时间差异通常被大幅减少的旋转开销所抵消。
	3.**性能的平衡性**
	红黑树在搜索、插入、删除这三种基本操作之间取得了极佳的平衡：如果你的应用场景是读多写少（如静态查表），	AVL 树更优。如果你的应用场景涉及大量的增删改查混合，红黑树的综合表现更加稳健。

**文件流操作**
文件的打开与关闭
fopen：打开一个文件，并返回文件指针。
fclose：关闭已打开的文件，释放资源。

二进制读写
fread：从文件中读取指定大小的数据块（通常用于结构体或数组）。
fwrite：将数据块写入到文件中。

字符与字符串读写
fgetc：从文件中读取一个字符。
fputc：向文件写入一个字符。
fgets：从文件中读取一行字符串。
fputs：向文件写入一个字符串。

格式化读写
fscanf：按照指定格式从文件中读取数据（类似 scanf）。
fprintf：按照指定格式将数据写入文件（类似 printf）。

状态检查
feof：检测是否已到达文件末尾（End Of File）。
ferror：检查文件操作过程中是否发生了错误。

位置定位与缓冲区
ftell：返回文件位置指针的当前位置（相对于文件起始处的字节偏移量）。
fseek：移动文件位置指针到指定位置。
rewind：将文件位置指针重置到文件的开头。
fflush：强制刷新缓冲区，将缓冲区内的数据立即写入文件。
